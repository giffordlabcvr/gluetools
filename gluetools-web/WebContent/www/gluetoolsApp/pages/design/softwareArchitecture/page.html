The Application Programming Interface (API) of GLUE is a rich, extensive set of commands for developing, querying and analysing its model.

The GLUE engine may be run using a command line interface (CLI), in which case the commands may be executed interactively or from files.

Alternatively the GLUE engine may be integrated into a web container, in which case the commands may be executed via a Representational State Transfer (REST) web service interactions.

GLUE is implemented in Java. However, it should not be viewed as a Java library, as it is not intended that other systems will interface directly with the Java code. The command set is the supported API, there is no Java API.

GLUE is database-centric. The GLUE Java engine relies on a Relational Database Management System (RDBMS) such as MySQL or Apache Derby. All aspects of a GLUE project are stored in this database.
