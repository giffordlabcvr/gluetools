Installing GLUEtools

You can run GLUEtools on any system which supports its prerequisites. This includes MS Windows, Linux and MacOSX.

If you are using Windows, we strongly recommend Cygwin, which you can download from cygwin.org.

Prerequisite: Java

Download and install Java 1.8.0 or later, from java.com.

Make sure you can run the correct version of the "java" program from the command line: 

$ java -version
java version "1.8.0_45"
Java(TM) SE Runtime Environment (build 1.8.0_45-b14)
Java HotSpot(TM) 64-Bit Server VM (build 25.45-b02, mixed mode)

Prerequisite: MySQL

GLUEtools requires a relational database manager. We recommend MySQL, certain other vendors are also possible.

Download and install MySQL 5.6 or later, from mysql.com.

GLUEtools connects to a named database with a specific username/password, and stores its data in that database. So within MySQL, you would normally create a username/password and named database specifically for GLUEtools. Example set up:

  • MySQL username: gluetools
  • MySQL password: glue12345
  • Database name: GLUE_TOOLS

You can use the following MySQL commands to set this up:

mysql> create user 'gluetools'@'localhost' identified by 'glue12345'; mysql> create database GLUE_TOOLS; mysql> grant all privileges on GLUE_TOOLS.* to 'gluetools'@'localhost';

Test that the new user/password works:

$ mysql -u gluetools --password=glue12345
Welcome to the MySQL monitor.  
Server version: 5.6.25 MySQL Community Server (GPL)

mysql>

In this MySQL session, test that the new named database works:

mysql> use GLUE_TOOLS; Database changed

Prerequisite: BLAST

GLUEtools uses the BLAST+ program for auto-alignment and certain other features.

Download and install BLAST+ 2.2.31 or later from NCBI's BLAST download page.

Download the GLUEtools jar

For command line use, GLUEtools is packaged as a self-contained Java jar. You can download it from the GLUEtools download page.

Create a GLUEtools configuration file

In a text editor, create an XML file gluetools-config.xml based on the XML below. GLUEtools will read this file each time it runs. Modify the XML file to specify:

  • Your MySQL username, password and database name
  • The location of the blast executables blastn and makeblastdb
  • GLUEtools creates BLAST database files. Specify a directory where GLUEtools can store these files

Download the GLUEtools example project

Download the example project from the GLUEtools download page.

Execute the following command line to unzip the project.

$ unzip exampleProject-0.1.34.zip

Make a script to run GLUEtools

Let's suppose you've put the jar and XML config file in /home/fred/gluetools. We suggest you create a bash script called gluetools.sh in /home/fred/gluetools/bin. The bash script contents should be:

#!/bin/bash java -jar ${GLUETOOLS_HOME}/gluetools-core-all-0.1.34.jar -c ${GLUETOOLS_HOME}/gluetools-config.xml ${@}
Make sure the bash script is executable: 
$ chmod u+x /home/fred/gluetools/bin/gluetools.sh
Add the following to your .bash_profile file

export GLUETOOLS_HOME=/home/fred/gluetools export PATH=$PATH:$GLUETOOLS_HOME/bin

Run GLUEtools

Now you can execute the following command line to start GLUEtools and build the example project

$ cd exampleProject
$ gluetools.sh -f exampleProject.glue

The -f option runs the commands in the exampleProject.glue file.

You will then see the GLUE interactive prompt.
