GLUEtools

Mission statement

A software package for developing, disseminating and utilising detailed hypotheses concerning the evolution of virus and EVE genomes.

Design overview

At the heart of GLUEtools is an entity-relationship model which has been designed for representing:
  • Virus and EVE genome data
  • Phenotypic and other data associated with specific sequences
  • Genomic feature annotations
  • The detailed proposed evolutionary relationships between genomes
  • Configurable modules for advanced manipulation and analysis of the data
Data hosted inside a GLUEtools instance is organised into independent projects. Typical activities associated with developing a GLUEtools project:
  • Import sequence data into GLUEtools from a public database or elsewhere
  • Define relevant additional sequence data fields and obtaining corrected / normalised values for these fields
  • Filter out irrelevant sequences
  • Select reference sequences and define genomic features on these references
  • Apply sequence alignment techniques to capture homologies between genome segments
  • Arrange data into evolutionary clades and define relationships between these clades
  • Define project-specific analysis modules
Once a project has been developed, it can be published as a web resource, which allows:
  • Dissemination of the project data, for example in conjunction with an academic publication
  • Utilisation of the project for automated sequence analysis

Software architecture

The API of GLUEtools is a rich, extensive set of commands for developing, querying and analysing its model.

The GLUEtools engine may be run in command line mode, in which case the commands may be executed interactively or from files.

Alternatively the GLUEtools engine may be integrated into a web container, in which case the commands may be executed via a REST-style web service interactions.

GLUEtools is implemented in Java. However, it should not be viewed as a Java library, as it is not intended that other systems will interface directly with the Java code. The command set is the supported API, there is no Java API.

GLUEtools is database-centric. The GLUEtools Java engine relies on a RDBMS such as MySQL or Apache Derby. All aspects of a GLUEtools project are stored in this database.
