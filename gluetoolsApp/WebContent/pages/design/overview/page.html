Core Schema

The central concept in GLUE is the Core Schema, a database design which captures:

  • Virus sequence data
  • Nucleotide-level homologies between sequences (alignments)
  • Groupings of sequences into clades and evolutionary relationships between clades
  • Genome features such as protein coding regions, annotated on reference sequences
  • Nucleotide or amino-acid patterns or motifs
  • Any project-specific auxiliary data, via the Schema Extensions mechanism

Developing GLUE Projects

Data and functionality inside GLUE is organised into independent self-contained Projects. Bioinformaticians can develop and make use of GLUE projects privately or collaboratively. A project focuses on a related set of virus sequences, for example:

  • Sequences within a particular virus species
  • Sequences within a specific outbreak
  • A subgenomic region of a given set of virus sequences

GLUE Projects on the Web

A GLUE project can also be published as a web resource by deploying GLUE as a Web Service. Example uses of this scenario:

  • Utilisation of the project functionality for automated sequence analysis, as part of an web-based tool aimed at reference lab scientists
  • Dissemination of the project data as an interactive resource, in conjunction with an academic publication

Bioinformatics Functionality

Users and client systems interact with the GLUE database and functionality via the Command Layer.

The core set of GLUE commands support operations such as:

  • Adding, deleting or modifying objects in the database
  • Interrogating the database using a rich query language
  • Deriving frequencies of amino acid resdiues within a certain set of sequences
  • Scanning sequences for specific nucleotide or amino acid patterns

Further GLUE command functionality is accessed using Modules.

These provide built-in functionality, customised on a project-specific basis.

GLUE contains an extensive range of module-based functionality including:

  • Importing sequences into the database from NCBI Nucleotide
  • Extracting and normalising data in a GenBank record
  • Computing new alignments
  • Assigning a genotype to sequences using a maximum-likelihood based method
  • Exporting sequences or alignments from the database for use outside GLUE
Users may also add their own project-specific functionality using the Scripting Layer.
